# DeepSeek-V4 提示词：Text Master 全页面设计图高还原实现

> 使用对象：DeepSeek-V4 / DeepSeek Coder / 具备项目读写能力的前端 Agent。  
> 目标：让 DeepSeek 根据根目录已解压的 Text Master 全页面设计图，尽可能实现与设计图一致的页面效果。  
> 设计图来源：项目根目录中已解压的 `TextMaster_all_pages_design_pack.zip`，应包含 `pages/` 目录与 14 张 PNG 设计图。

---

## 0. 最重要的架构前提

请严格遵守以下产品架构，不要误解：

```text
Text Master 是独立 App。
Brain Hub 是类似 Steam 的启动器 / 应用库 / 分发平台 / 可选同步入口。
Text Master 不依赖 Brain Hub 才能运行。
Brain Hub Ready 只能表示“可选同步已准备”，不能作为运行前置条件。
```

禁止做成：

```text
/apps/text-master 作为唯一运行入口
Text Master 必须等待 Brain Hub 上下文才能启动
首页像 Brain Hub 子页面
```

正确做法：

```text
Text Master 自己有首页、项目中心、新建项目、工作台、模板中心、导出中心、设置、用户资料。
Brain Hub 相关能力只以状态胶囊、可选同步、Adapter 占位形式出现。
```

## 2. 总执行提示词

把下面整段作为 DeepSeek-V4 的总任务提示词：

```text
你是一个资深前端工程师和 UI 还原专家。请读取项目根目录下的 Text Master 全页面设计图，按设计图实现 Text Master 独立 App 的所有页面。

目标：实现与设计图肉眼一致度 95% 以上的页面效果。重点不是只做功能，而是把布局、视觉层级、按钮位置、卡片尺寸、深色风格、状态胶囊、项目卡片、工作台三栏结构尽量还原。

严格要求：
1. Text Master 是独立 App，不依赖 Brain Hub 存在。
2. Brain Hub Ready 只是可选同步状态，不是运行前置条件。
3. 首页不展示“文本生产链路”卡片。
4. 首页右侧主位置放“快速模板”。
5. 首页原快速模板下方位置放“广告位 / 官方比赛”。
6. 顶部保留用户资料按钮。
7. 顶部不能出现重复的新建项目按钮。
8. 新建项目按钮只在首页 Hero 区作为主按钮出现。
9. 所有页面保持深色专业工具软件风格，不允许大面积白底。
10. 所有页面在 1440×900 和 1366×768 下不能横向溢出。
11. 不要引入大型 UI 库，优先使用项目已有技术栈和轻量 CSS。
12. 不要破坏现有路由，不要自动 git commit，不要自动 git push。

请按以下顺序完成：
1. 读取设计图并建立页面对应表。
2. 建立或统一全局样式变量。
3. 实现顶部导航、卡片、按钮、状态胶囊、工作台三栏等通用组件。
4. 按设计图逐页还原首页、新建项目、工作台各子页、模板、导出、设置、用户资料。
5. 运行 typecheck/build。
6. 运行视觉审计脚本，输出所有页面截图和 zip。
7. 若页面总分低于 95，修复明显问题后再跑一次审计。

最后输出：修改文件列表、截图路径、审计 zip 路径、各页面评分、总分、未完成问题。
```
